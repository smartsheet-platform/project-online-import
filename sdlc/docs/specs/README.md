# Spec Mode Guidance Directory

This directory contains guidance documents that the project online Spec mode references for project-specific specification standards, templates, and examples.

## What to Add Here

### Specification Templates
- **specification-template.md** - Your project's standard specification format
- **api-spec-template.md** - Template for API integration specifications
- **feature-spec-template.md** - Template for feature specifications
- **component-spec-template.md** - Template for component specifications

### Documentation Standards
- **documentation-standards.md** - Your project's documentation style guide and requirements
- **specification-patterns.md** - Common specification patterns and approaches used
- **review-checklist.md** - Checklist for specification review and validation

### Examples
- **good-specification-examples.md** - Examples of well-written specifications from your project
- **api-integration-examples.md** - Examples of API integration specifications
- **error-handling-examples.md** - Examples of error handling documentation
- **user-workflow-examples.md** - Examples of user workflow documentation

### Project-Specific Guidance
- **domain-vocabulary.md** - Project-specific terminology and domain language
- **compliance-requirements.md** - Regulatory or compliance requirements for documentation
- **stakeholder-guidelines.md** - Guidelines for different stakeholder audiences

## How Spec Mode Uses These Files

The project online Spec mode will:
1. **Reference these templates** when creating specifications
2. **Follow documentation standards** defined here  
3. **Use project-specific terminology** and patterns
4. **Ensure consistency** across all project specifications
5. **Meet quality standards** defined in your guidelines

## Template Instructions

1. **Replace project online** with your actual project name
2. **Create the files listed above** with your project's actual standards
3. **Include real examples** from your existing specifications
4. **Document your specific requirements** and compliance needs

The Spec mode references these files to understand your project's documentation standards and create specifications that match your quality and style requirements.
